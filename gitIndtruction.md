# Инструкция по GIT. Часть 1.

Данная инструкция рассчитана на человека, прежде не работавшего с системами контроля версий, однако, возможно, имеющего представление о том, что такое командная строка. Здесь будут описаны только самые базовые операции.

## Введение

Системы контроля версий, одной из которых является git, используются при работе с самыми разными проектами, начиная от программных проектов и заканчивая написанием книги или статьи. При работе с документами мы часто пользуемся командами CTRL+S *(сохранить)* и CTRL+Z *(отменить)*, а если мы хотим сделать несколько вариантов, нам приходится сохранять каждый из них в отдельный файл, что не всегда удобно. Git же позволяет делать все то же самое, но не на уровне отдельного файла, а на уровне целого проекта. Каталог с проектом в этом случае называется репозиторием **(repository)**. Каждое такое глобальное CTRL+S называется коммитом **(commit)** и сохраняет текущую версию проекта, к которой всегда можно будет вернуться в любой момент (глобальный CTRL+Z). Также возможна работа с несколькими версиями, при этом не возникает множества файлов разных версий *(на самом деле все изменения сохраняются в системной папке .git, находящейся в корне репозитория, и ее лучше не трогать)*. Кроме того git позволяет работать с несколькими копиями репозитория, беря на себя все проблемы, связанные с синхронизацией данных. Это не только предохраняет от потери данных, но и делает возможным эффективную командную работу над проектом.

## Установка GIT

1. Скачать GIT с [официального сайта](https://git-scm.com/downloads).
2. Установить GIT на компьютер.
3. Представиться с помощью команд:
    
    *  git config ­­--global user.name "Ваше имя"
    *  git config ­­--global user.email "Ваша почта"

Теперь можно начинать работу!

## Начало работы

Для начала создадим папку, в которой будем работать, и откроем её через VSCode. Создадим в ней файл, который GIT будет отслеживать. Например: "example.md" *(расширение может быть любым)*.

Теперь откроем терминал. Это можно сделать через вкладку "Вид" -> "Терминал".

Для работы с GIT нам нужно освоить базовые команды.

## Базовые команды

* git init - инициализация локального репозитория
* git status - получить информацию от git о его текущем состоянии
* git add - добавить файл или файлы к следующему коммиту
* git commit -m "message" - создание коммита
* git log - вывод на экран истории всех коммитов с их хеш-кодами
* git checkout - переход от одного коммита к другому
* git checkout master - вернуться к актуальному состоянию и продолжить работу
* git diff - увидеть разницу между текущим файлом и закоммиченным файлом

# Инструкция по работе с GIT. Часть 2.

## Работа с ветками

Работа с ветками в GIT может понадобиться, когда мы хотим попробовать что-то, но не хотим затрагивать чистовой файл, или при работе в команде. Например, можно создать по ветке на каждого участника проекта, чтобы он работал именно в ней. Когда части проекта будут готовы, можно будет просто слить ветки с веткой master и получить готовый проект.

Для начала разберём основные команды для работы с ветками.

### Основные команды

* git branch - позволяет вывести в терминал список всех веток и укажет, на какой ветке  данный момент Вы находитесь
* git branch branchName - команда для создания новой ветки. Имя ветки может быть любым, но на английском и без пробелов
* git checkout branchName - позволяет перейти на ветку по её названию и продолжать работу в ней
* git merge branchName - позволяет слить ветку, имя которой указано, с веткой мастера
* git branch -d branchName - позволяет удалить уже слитую ветку
* git log --graph - позволяет вывести в терминал лог коммитов с визуализацией между ними

## Конфликты в GIT

Когда в разных ветках в одних и тех же строках содержится разная информация, при слиянии этих веток происходит конфликт. GIT не модет самостоятельно разрешить этот конфликт и предлагает сделать это пользователю. Давайте посмотрим, что же именно происходит. Для этого попробуем слить с веткой мастера ветку с текстом.

![слияние](/gitHomeWork/images/Снимок21.PNG)

Итак, мы видим два окна. Зеленое окно - это та информация, которая содержится на строках в ветке мастера. В синем окне на этих же строках содержится совершенно другая информация. Также мы видим сверху кнопки вариантов того, что мы можем сделать.

* Принять текущее изменение - значит оставить информацию только с ветки мастера, на которой мы сейчас находимся.
* Принять входящее изменение - значит оставить только ту информацию, которая содержится в ветке с текстом (которую мы сливаем с веткой мастера).
* Принять оба изменения - значит оставить информацию с обоих веток. При этом входящее изменение будет находиться под текущим.
* Сравнить изменения - позволяет наглядно посмотреть, чем отличается информация в разных ветках.

Выберем "Принять оба изменения", так как нам нужно сохранить информацию с обоих веток.

Таким образом сольём оставшиеся ветки с веткой мастера, разрешив все всплывающие конфликты. Не забываем после каждого слияния коммитить изменения в файле.

В итоге мы получим готовый файл с информацией со всех веток. УРА!

![результат](/gitHomeWork/images/Снимок22.PNG)

## Работа с изображениями в GIT (Дополнительно)

Когда мы добавили в наш репозиторий картинки, команда git status сообщила нам о том, что они не отслеживаются. Давайте исправим это.

Картинки не добавляются в отслеживаемые файлы с помощью команды git add. Вместо этого в репозитории создаётся файл ".gitignore", в котором прописываются названия картинок.

# Инстркция по GIT. Часть 3.

## Работа с pull request

Работа в GIT часто связана с командной работой. Чтобы нескольким людям легче было вносить изменения в проект, был создан GitHub. 

Для начала нужно зарегистирироваться. Затем можно найти проект, над которым вы хотели бы поработать (помочь другим людям с работой над проектом). В нашем случае мы переходим по ссылке на проект.

Например: https://github.com/maxim-stoma/gitHomeWork

1. Первое, что мы сделаем, нажмём на кнопку "Fork" и создадим копию этого репозитория у себя на аккаунте.

![fork](/gitHomeWork/images/Снимок27.PNG)

2. Теперь переходим в наш проводник, создаём новую папку, открываем её в VSCode и проверяем командой git status, чтобы эта папка не была репозиторием. После этого переходим в GitHub, копируем указанную ссылку и пишем в терминале команду git clone <указанная ссылка>.

![ссылка](/gitHomeWork/images/Снимок28.PNG)

![git_clone](/gitHomeWork/images/Снимок29.PNG)

3. Теперь создадим отдельную ветку, например, со своим именем, и перейдём на неё.

![новая_ветка](/gitHomeWork/images/Снимок30.PNG)

4. Создадим новый файл, куда будем добавлять нашу информацию. В нашем случае это файл "gitInstruction.md". Так же не забудем его добавить и закоммитить.

![новый_файл](/gitHomeWork/images/Снимок31.PNG)

5. После того, как вся информация добавлена, нужно снова закоммитить изменения. После этого введём в терминал команду "git push". Терминал немного исправляет нашу команду. Копируем, вставляем и нажимаем "ввод".

6. Теперь переходим в GitHub. Обновляем страницу и видим, что нам сообщают о некоторых изменениях в файле. Нажимаем на кнопку "Create and pullrequest".

Обязательно заполним описание того, что мы добавили. Затем отправляем наш запрос владельцу репозитория. В свою очередь он может либо принять, либо не принять изменения. Всё будет зависеть от его мнения)
